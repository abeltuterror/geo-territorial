R-tree — Índice espacial

Cómo rbush encuentra puntos dentro de un radio en O(log n) en lugar de O(n)

1
El problema — iteración lineal

Sin índice, para filtrar puntos dentro de 3 km hay que medir la distancia a los 18 763 puntos uno por uno. Eso es O(n) — con 18k puntos son 18 763 operaciones de distancia por cada búsqueda.

El R-tree evita casi todas esas comparaciones organizando los puntos en una jerarquía de rectángulos. Una búsqueda descarta regiones enteras del espacio sin mirar los puntos individuales.

Sin R-tree — O(n)
18 763 pts
Comparaciones
18 763 distancias
Con R-tree — O(log n)
18 763 pts
Comparaciones
~log₂(18763) ≈ 14 nodos
2
Cómo se estructura — cajas dentro de cajas

El R-tree agrupa puntos cercanos dentro de bounding boxes (MBR) — el rectángulo más pequeño que contiene a todos sus hijos. Esas cajas se anidan en niveles:

Nodo raíz (toda el área)
Nodos internos (grupos)
Hojas (grupos pequeños)
Puntos individuales
Cada caja contiene varios puntos. Nunca se solapan innecesariamente.

4
El código en el proyecto

Así se usa rbush en geo.worker.ts:104:

// 1. Cargar todos los puntos en el índice (una vez por búsqueda)
const tree = new RBush()
tree.load(points.map(p => ({
  minX: p.longitud, minY: p.latitud,
  maxX: p.longitud, maxY: p.latitud,   // punto = caja de tamaño 0
  id: p.id
})))

// 2. Convertir radio (km) a grados y buscar el cuadrado
const delta      = radiusKm / 111      // 3km → 0.027°
const candidates = tree.search({      // O(log n) — devuelve candidatos
  minX: center[0] - delta,  maxX: center[0] + delta,
  minY: center[1] - delta,  maxY: center[1] + delta,
})

// 3. Descartar esquinas con distancia exacta (solo sobre candidatos)
const filtered = candidates.filter(c => {
  const dist = turf.distance(centerPt, turf.point([c.longitud, c.latitud]))
  return dist <= radiusKm
})
5
Por qué dos pasos — cuadrado + círculo

El R-tree solo sabe buscar rectángulos, no círculos. Por eso el paso 2 devuelve los puntos dentro del cuadrado, que incluye las esquinas. El paso 3 aplica la distancia real de Turf.js solo sobre ese subconjunto pequeño para cortar las esquinas.

Los puntos amarillos pasan el bbox pero fallan la distancia exacta — son descartados en el paso 3.
