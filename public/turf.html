2. turf.buffer() — Expandir polígono
geo.worker.ts : 48 buildPolygon()

Expande el convex hull 50 metros hacia afuera. Sin el buffer, los puntos del borde del territorio quedarían exactamente en el filo del polígono y visualmente parecería que están fuera. El buffer los deja claramente dentro.

Sin buffer

Los puntos del borde tocan el filo del polígono.

Con buffer (0.05 km)

El polígono se expande 50 m — todos los puntos quedan dentro con margen.

// geo.worker.ts:48
return turf.buffer(hull, 0.05, { units: "kilometers" })

3. turf.distance() — Distancia exacta
geo.worker.ts : 133 FILTER_RADIUS

Calcula la distancia real en kilómetros entre dos coordenadas usando la fórmula Haversine — que tiene en cuenta la curvatura de la Tierra. Se usa en el filtro de radio para descartar los puntos que pasaron el bbox del R-tree pero están fuera del círculo real.

Centro (búsqueda)
Dentro del radio ✓
Fuera del radio ✗ (pasó el bbox)
Fuera del bbox
turf.distance() solo se llama sobre los puntos amarillos candidatos del R-tree — no sobre los 18 763.
// geo.worker.ts:129
const centerPt = turf.point(center)

const filtered = candidates.filter(c => {
  const dist = turf.distance(
    centerPt,
    turf.point([c.longitud, c.latitud]),
    { units: "kilometers" }
  )
  return dist <= radiusKm   // ← descarta esquinas del bbox
})
Distancia euclidiana (incorrecta)
√((x2-x1)² + (y2-y1)²)

Trata las coordenadas como un plano plano. En Lima introduce error de ~2–5 km en distancias de 10 km.

Haversine — turf.distance()
2r·arcsin(√(sin²(Δlat/2)+cos·sin²(Δlon/2)))

Tiene en cuenta la curvatura de la Tierra. Resultado exacto en km independiente de la latitud.

4. turf.point() y turf.featureCollection() — Convertir datos
GeoJSON helpers geo.worker.ts

Turf trabaja con objetos GeoJSON. Estas dos funciones convierten los datos del proyecto al formato que Turf entiende.

// SalesPoint → GeoJSON Point
turf.point([p.longitud, p.latitud])
// → { type: "Feature", geometry: { type: "Point", coordinates: [-77.03, -12.04] } }

// Array de puntos → GeoJSON FeatureCollection
turf.featureCollection([pt1, pt2, pt3, ...])
// → { type: "FeatureCollection", features: [...] }
// requerido por turf.convex()
