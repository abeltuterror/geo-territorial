Web Worker — workers/geo.worker.ts

Procesamiento pesado fuera del hilo principal de JavaScript

¿Por qué un Web Worker?

JavaScript es single-threaded: si un cálculo tarda 800 ms en el hilo principal, el navegador no puede pintar ni responder a clicks durante ese tiempo — la UI se congela.

El algoritmo de asignación construye 18 763 × k pares (hasta 1,6 M entradas), los ordena y los recorre en greedy. Con 90 vendedores eso son cientos de ms de cómputo puro. Moverlo a un Worker libera el hilo principal completamente: el mapa sigue a 60 fps y los indicadores de estado siguen animándose mientras se calcula.

El worker se crea al montar el hook con import.meta.url — Webpack 5 (incluido en Next.js) detecta este patrón y empaqueta el worker en un bundle separado automáticamente. Se termina en el cleanup del useEffect.

useEffect(() => {
  workerRef.current = new Worker(
    new URL("../workers/geo.worker.ts", import.meta.url),
    { type: "module" }   // ← permite imports ES dentro del worker
  )

  workerRef.current.onmessage = (e) => {
    const { type, payload } = e.data
    if (type === "TERRITORIES_READY") saveTerritoriesFromWorker(payload)
    if (type === "FILTER_READY") {
      const ids = new Set(payload.map(p => p.id))
      setState((prev) => ({ ...prev, filteredIds: ids, isProcessing: false }))
    }
  }

  return () => workerRef.current?.terminate()   // ← limpieza al desmontar
}, [])
Algoritmo 1 — ASSIGN_TERRITORIES
Greedy Nearest Neighbor geo.worker.ts : 55

Asigna exactamente pointsPerSeller puntos a cada vendedor seleccionado sin solapamiento. No usa K-means porque K-means no tiene restricción de capacidad y produce clusters de tamaños desiguales.

1
Semillas aleatorias — pickRandomSeeds(points, k) elige k puntos reales del dataset como centros iniciales. Usa un Set para garantizar que no se repitan.
2
Tabla de pares — doble bucle O(n×k) que calcula la distancia euclidiana al cuadrado entre cada punto y cada semilla. Con 18 763 puntos y 10 vendedores = 187 630 pares.
3
Ordenación — pairs.sort() por distancia ascendente. O(n·k · log(n·k)). El par más corto queda primero.
4
Greedy con cupo — recorre los pares ordenados. Un punto se asigna al primer vendedor que todavía tiene cupo. Si el punto ya fue tomado o el vendedor está lleno, se salta. Termina cuando todos los vendedores llegan a su cupo.
5
Polígono — buildPolygon() llama a turf.convex() sobre los puntos asignados y le aplica un turf.buffer(0.05 km) para que el borde no corte puntos de borde.
// paso 2 y 3 — pares + ordenación
const pairs = []
for (let pi = 0; pi < points.length; pi++) {
  for (let si = 0; si < k; si++) {
    const d = (points[pi].longitud - seeds[si].longitud) ** 2
             + (points[pi].latitud  - seeds[si].latitud)  ** 2
    pairs.push({ pi, si, dist: d })
  }
}
pairs.sort((a, b) => a.dist - b.dist)

// paso 4 — greedy con capacidad
const pointTaken    = new Set()
const sellerCapacity = new Array(k).fill(pointsPerSeller)

for (const { pi, si } of pairs) {
  if (pointTaken.has(pi)) continue       // ya asignado
  if (sellerCapacity[si] <= 0) continue  // vendedor lleno
  pointTaken.add(pi)
  sellerCapacity[si]--
  sellerPoints[si].push(points[pi])
}

Algoritmo 2 — FILTER_RADIUS
R-tree (rbush) geo.worker.ts : 104

Filtra los puntos dentro de un radio en km desde un centro dado. Se activa cada vez que el usuario selecciona una ubicación en la barra de búsqueda.

1
Construye el R-tree — tree.load(points) inserta los 18 763 puntos en el índice espacial de rbush. Bulk-load es O(n log n).
2
Bounding box aproximado — convierte el radio en grados (delta = radiusKm / 111) y consulta el R-tree con ese bbox. Devuelve solo los candidatos dentro del cuadrado — O(log n).
3
Distancia exacta — sobre los candidatos del bbox aplica turf.distance() para descartar los que quedan en las esquinas del cuadrado pero fuera del círculo real.
// paso 1 — índice R-tree
const tree = new RBush()
tree.load(points.map(p => ({
  minX: p.longitud, minY: p.latitud,
  maxX: p.longitud, maxY: p.latitud,
  id: p.id
})))

// paso 2 — bbox O(log n)
const delta      = radiusKm / 111
const candidates = tree.search({
  minX: center[0] - delta, minY: center[1] - delta,
  maxX: center[0] + delta, maxY: center[1] + delta,
})

// paso 3 — distancia exacta solo sobre candidatos
const filtered = candidates.filter(c => {
  const dist = turf.distance(centerPt, turf.point([c.longitud, c.latitud]))
  return dist <= radiusKm
})
