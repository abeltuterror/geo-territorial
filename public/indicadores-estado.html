Indicadores de Estado

Los tres estados de carga del sistema y cómo se comunican al usuario

Estado 1 — Carga inicial
● Bloqueante page.tsx : 38

Al montar la app, useGeoData dispara tres fetch en paralelo con Promise.all. Mientras no terminan los tres, isLoading vale true. page.tsx detecta ese valor y devuelve una pantalla completa en lugar del mapa — la UI entera está bloqueada hasta que los datos llegan.

Trigger
fetch("/api/points"), fetch("/api/sellers"), fetch("/api/territories") — Promise.all
→
Efecto visual
Pantalla completa reemplaza toda la UI
// useGeoData.ts — estado inicial
const [state, setState] = useState({
  isLoading: true,   // ← arranca en true
  ...
})

// Al terminar los tres fetch:
setState((prev) => ({ ...prev, isLoading: false }))
// page.tsx:38 — cortocircuito
if (isLoading) {
  return (
    <div className="w-screen h-screen flex items-center justify-center">
      <Loader2 className="w-10 h-10 text-blue-400 animate-spin" />
      <p>Cargando 18,763 puntos de venta...</p>
    </div>
  )
}  // ← el resto del componente nunca ejecuta
pantalla completa — mapa no renderizado aún

Estado 2 — Worker procesando
● No bloqueante page.tsx : 76

Cuando el usuario pulsa "Generar territorios", el mensaje se envía al Web Worker. isProcessing pasa a true mientras el worker calcula el clustering en su hilo separado. El mapa sigue visible e interactivo — solo aparece un chip semitransparente en la barra superior del mapa indicando que hay cálculo en curso.

Trigger
worker.postMessage({ type: "ASSIGN_TERRITORIES" })
→
Efecto visual
Chip con spinner sobre el mapa (no bloquea)

Estado 3 — Botón deshabilitado
● No bloqueante TerritoryPanel.tsx : 88

El mismo flag isProcessing llega como prop a TerritoryPanel. Mientras vale true, el botón "Generar territorios" se deshabilita con disabled, cambia de color a gris y su texto muta a "Procesando..." para evitar que el usuario envíe un segundo cálculo mientras el primero no terminó.

Trigger
isProcessing === true (prop desde page.tsx)
→
Efecto visual
Botón gris + disabled + texto "Procesando..."

Estado 4 — Hydration de GeoMap
● WebGL page.tsx : 12

Deck.gl usa WebGL y no puede ejecutarse en el servidor (SSR). Por eso GeoMap se importa con dynamic() y ssr: false. Mientras Next.js descarga e inicializa el bundle del mapa en el cliente, el loading de dynamic() muestra un spinner en el área del mapa. Este estado es independiente de los datos — ocurre por la carga del bundle JS. 

// page.tsx:12 — import dinámico sin SSR
const GeoMap = dynamic(
  () => import("@/components/map/GeoMap"),
  {
    ssr: false,                     // ← WebGL no existe en Node
    loading: () => (
      <div className="w-full h-full flex items-center justify-center">
        <Loader2 className="w-8 h-8 text-blue-400 animate-spin" />
      </div>
    ),
  }
)

Resumen — los 4 indicadores

Flag / mecanismo Qué espera Dónde aparece Bloquea UI
