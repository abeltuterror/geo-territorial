3. Hook central Hook

Hace el fetch a /api/points y guarda el resultado en estado React. También gestiona la comunicación con el Web Worker y expone filteredIds.

const [points, setPoints] = useState<SalesPoint[]>([])

useEffect(() => {
  fetch('/api/points')
    .then(r => r.json())
    .then(data => setPoints(data.points))
}, [])

5. GeoMap — prepara los datos React

visiblePoints filtra los puntos según filteredIds. territoryColorMap construye un Map<id, [r,g,b]> para saber qué color asignar a cada punto.

const visiblePoints = useMemo(() =>
  filteredIds ? points.filter(p => filteredIds.has(p.id)) : points
, [points, filteredIds])

const territoryColorMap = useMemo(() => {
  const map = new Map()
  territories.forEach(t =>
    t.puntosVenta.forEach(sp => map.set(sp.id, hexToRgb(t.color)))
  )
  return map
}, [territories])
↓
GeoMap.tsx
Deck.gl → GPU
6. ScatterplotLayer — renderizado GPU WebGL

Recibe visiblePoints como data. Deck.gl sube los datos a la GPU y ejecuta getFillColor y getPosition por cada punto en paralelo — el hilo principal de JS no itera nada.

new ScatterplotLayer({
  data: visiblePoints,              // ← los puntos llegan aquí
  getPosition: d => [d.longitud, d.latitud],
  getFillColor: d => {
    const color = territoryColorMap.get(d.id)
    return color
      ? [...color, 220]         // color del territorio
      : [160, 160, 160, 180]    // gris si no tiene territorio
  },
  updateTriggers: { getFillColor: [territoryColorMap] }
})
